Time extractor

Программа для расчёта активного времени сотрудников 2-й линии в чатах1С-Коннект.

Что умеет

Загружать JSON-файл с диалогами.

Загружать данные из HTTP-сервиса 1С через proxy.

Определять сотрудников 2-й линии по компании автора.

Позволять выбрать сотрудников для расчёта.

Считать активное время по циклам «вопрос 1-й линии — первый ответ 2-й линии».

Показывать результат по сотрудникам.

Открывать читабельные диалоги, вошедшие в расчёт.

Экспортировать результат в CSV.

Загружать тикеты из .xls и .xlsx.

Считать время решения завершённых тикетов за вычетом пауз.

Считать фактическую длительность работы по тикетам.

Показывать статистику по сотрудникам, приоритетам и видам работ.

Строить матрицу «Вид работ × Приоритет».

Складывать время работы в чатах и заполненную длительность тикетов.

Правила расчёта

Программа считает не фактическое время за компьютером, а расчётное активноевремя работы с обращениями.

Системные сообщения исключаются.

Последовательные сообщения 1-й линии образуют один блок вопроса.

Точка отсчёта — последнее сообщение 1-й линии перед ответом.

Учитывается только первый последующий ответ 2-й линии.

Последующие сообщения 2-й линии не создают новые интервалы, пока сноване напишет 1-я линия.

Если интервал меньше 10 секунд, засчитывается 10 секунд.

Если интервал составляет от 10 секунд до 15 минут включительно,засчитывается фактический интервал.

Если интервал превышает 15 минут, время не засчитывается: такой вопросдолжен обрабатываться в системе тикетов.

Сообщения 2-й линии без предшествующего вопроса 1-й линии не учитываются.

Настройки по умолчанию:

максимальный активный интервал — 15 минут;

минимальное время на ответ — 10 секунд;

компания сотрудников — ООО Селекта Сервис.

Правила расчёта тикетов

В расчёт включаются только тикеты со статусом Завершена. Выполняющиеся,отменённые и незавершённые тикеты исключаются.

Время решения:

Заявка закрыта − Заявка открыта − Длительность пауз

Фактическое рабочее время берётся из поля Длительность. Значение 0:00считается незаполненным и не участвует в расчёте среднего фактического времени.

Для загрузки обязательны поля:

№;

Статус;

Приоритет;

Вид работ;

Исполнитель;

Длительность;

Заявка открыта;

Заявка закрыта.

Поле Длительность пауз необязательно. Если его нет, программа показываетпредупреждение и принимает паузы равными нулю.

Статистика строится:

по сотрудникам;

по приоритетам;

по видам работ;

в матрице «Вид работ × Приоритет»;

в детализации завершённых тикетов.

Общее учтённое время сотрудника:

Расчётное время в чатах + заполненная длительность тикетов

Время от регистрации до завершения тикета в загрузку сотрудника не добавляется.

Для чтения старого формата Excel используется локальный файлxlsx.full.min.js.

Локальный запуск

Открыть папку проекта в VS Code.

Запустить proxy:

node server.js

Открыть index.html через Live Server.
